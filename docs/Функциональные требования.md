## Функциональные требования

### 1. Аутентификация и доступ

Система должна поддерживать аутентификацию пользователей по JWT.

Система должна обеспечивать доступ пользователя только к тем данным и операциям, на которые у него есть права.

---

### 2. Воркспейсы

Система должна поддерживать два типа воркспейсов:

- приватные;
- общие.

Приватный воркспейс доступен только его владельцу.

Общий воркспейс доступен пользователям, которым он назначен, либо всем пользователям системы — в зависимости от настроек доступа.

Пользователь должен видеть список доступных ему воркспейсов:

- свои приватные;
- общие, к которым у него есть доступ.

---

### 3. Роли в воркспейсе

Система должна поддерживать роли пользователя в воркспейсе:

- Администратор;
- Пользователь.

Администратор воркспейса может:

- изменять системный промпт воркспейса;
- добавлять документы в воркспейс;
- удалять документы из воркспейса;
- управлять составом участников общего воркспейса и их ролями.

Пользователь воркспейса может:

- работать с чатами в рамках доступного воркспейса;
- отправлять сообщения нейросети;
- получать ответы;
- просматривать историю своих чатов.

---

### 4. Системный промпт воркспейса

Для каждого воркспейса система должна хранить отдельный системный промпт.

Система должна использовать системный промпт воркспейса при обработке всех запросов пользователя в рамках этого воркспейса.

Изменять системный промпт воркспейса может только Администратор.

---

### 5. Документы воркспейса

Каждый воркспейс должен иметь собственный набор документов.

Документы воркспейса должны использоваться как контекстная база знаний для ответов нейросети в рамках этого воркспейса.

Добавлять и удалять документы воркспейса может только Администратор.

Система должна обеспечивать изоляцию документов между воркспейсами.

---

### 6. Чаты

Чат является личной сущностью пользователя в рамках воркспейса.

Чат не является общим для участников воркспейса и не должен быть доступен другим пользователям.

Пользователь должен иметь возможность создавать неограниченное количество чатов в доступном ему воркспейсе.

Каждый чат должен принадлежать:

- одному пользователю;
- одному воркспейсу.

Чат должен использоваться как контекст взаимодействия пользователя с нейросетью.

---

### 7. Сообщения и история

Пользователь должен иметь возможность отправлять сообщения в рамках своего чата.

Система должна сохранять историю сообщений в каждом чате.

При повторном открытии чата система должна восстанавливать историю сообщений этого чата.

История чата должна быть доступна только владельцу этого чата.

---

### 8. Взаимодействие с нейросетью

При обработке сообщения система должна использовать:

- системный промпт воркспейса;
- документы воркспейса;
- историю текущего чата.

Система должна формировать ответ только в контексте текущего воркспейса и текущего чата.

Система не должна использовать данные других воркспейсов или чужих чатов.

---

### 9. Бизнес-правила

Каждый пользователь может иметь доступ одновременно к нескольким воркспейсам.

В одном воркспейсе один пользователь может иметь неограниченное количество личных чатов.

Чаты разных пользователей внутри одного воркспейса изолированы друг от друга.

Документы, системный промпт и роли являются сущностями воркспейса.

Чаты и история сообщений являются личными сущностями пользователя внутри воркспейса.
