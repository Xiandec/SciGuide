# Paper

Самодостаточная папка для курсовой, отчета или ВКР внутри `docs/kr`. Ее можно переносить между репозиториями целиком: текст, приложения, ресурсы, шаблон и сборка лежат в одном месте.

## Структура

- `manuscript.md` — основной текст работы.
- `appendices/*.md` — приложения; один файл равен одному приложению.
- `assets/` — изображения и другие файлы, на которые ссылается текст.
- `metadata/` — шаблон, библиография, фильтры и скрипт сборки.
- `output/paper.pdf` — результат рендера.
- `render.sh` — единая точка входа для сборки.

## Как писать

- Основной текст редактируй только в `manuscript.md`.
- Заголовок `#` — структурный раздел с новой страницы. `##` — подраздел внутри главы.
- Источники указывай как `[@ключ]`; сами записи лежат в `metadata/references.bib`.
- Рисунок вставляй так: `![Подпись](assets/image.png){#fig-name width=88%}`.
- Таблицу оформляй как markdown-таблицу и строку `Table: Подпись {#tbl-name}` сразу после нее.
- Листинг оформляй fenced-кодом с атрибутами, например: ```` ```python {#lst-example caption="Подпись"} ````.
- Ссылки на рисунки, таблицы и листинги делай так: `` `#ref(<fig-name>)`{=typst} ``.
- Каждое приложение пиши отдельным файлом в `appendices/`. Заголовок внутри файла станет названием приложения.

## Как настроить

- Тему, автора, группу, руководителя и год меняй в `metadata/metadata.yaml`.
- Правила оформления и титульный лист меняй в `metadata/template.typ`.
- Библиографический стиль лежит в `metadata/csl/`.

## Как рендерить

- Из корня репозитория: `./docs/kr/paper/render.sh`
- Из папки `docs/kr/paper`: `./render.sh`
- Сборка в другой файл: `./render.sh output/my-work.pdf`

Для рендера нужны установленные `pandoc` и `typst`.
