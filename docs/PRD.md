# PRD / описание проекта SciGuide

## 1. Название проекта

**SciGuide** — автоматическая справочная система по выбранному научному направлению на основе языковых моделей, семантического поиска и графового перерангирования.

## 2. Контекст и проблема

При работе с научными материалами пользователь сталкивается с двумя типовыми проблемами. Во-первых, классический keyword search плохо находит фрагменты, где нужная идея выражена другими формулировками. Во-вторых, даже семантический поиск по эмбеддингам не всегда учитывает структурные связи между понятиями: какие термины часто встречаются вместе, какие темы логически следуют друг за другом, какие концепты образуют устойчивые локальные кластеры в корпусе.

Из-за этого в top-k выдаче могут оказываться семантически похожие, но контекстно слабые фрагменты, тогда как более полезные куски текста теряются, если их релевантность проявляется не только через близость к запросу, но и через положение в сети понятий предметной области. Для научной справочной системы это особенно критично, потому что ответы часто требуют не одного фрагмента, а согласованного набора связанных фрагментов.

## 3. Цель проекта

Разработать прототип автоматической справочной системы, которая отвечает на вопросы по выбранному научному направлению, используя двухэтапный retrieval pipeline:

1. первичный поиск релевантных текстовых чанков по векторному семантическому сходству;
2. дополнительное перерангирование найденных кандидатов с учетом графа связей между понятиями, извлеченными из корпуса.

## 4. Идея решения

Базовый слой системы — обычный semantic search по эмбеддингам документов и пользовательского запроса. Он дает хороший recall и позволяет быстро получить ограниченный набор кандидатов.

Поверх него строится графовая модель корпуса. Из каждого чанка текста эвристически выделяются понятия. Между понятиями создаются связи двух типов:

- по совместному появлению в одном чанке;
- по относительному порядку появления в чанке или в локальном текстовом окне.

Далее запрос тоже проецируется в пространство понятий: из него выделяются опорные термины, после чего по графу рассчитывается структурная значимость близких узлов. Эта значимость используется для перерасчета score чанков. В итоге итоговая релевантность зависит не только от близости эмбеддингов, но и от того, насколько чанк покрывает важные узлы и связи в окрестности запроса.

Такой подход соответствует современному направлению graph-guided retrieval: граф не заменяет векторный поиск, а уточняет его, добавляя relational signal.
